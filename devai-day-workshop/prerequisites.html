Workshop Prerequisites

Please complete this setup before the workshop so we can spend the day building, not installing. Start with the core requirements, then check the extras for the paths you plan to follow.

Core Prerequisites (everyone)

Required for every path — set these up first

  • GitHub account — created and signed in. Sign up
  • GitHub Copilot license — Individual, Business, or Enterprise, activated on your account. Check plans & access
  • Visual Studio Code (latest version) installed. Download VS Code
  • Copilot extensions for VS Code — install GitHub Copilot and GitHub Copilot Chat, then sign in with your GitHub account. Get the extension
  • Git installed and configured. Install Git
  • A modern browser and a stable internet connection (some exercises run on github.com and in the cloud).
Confirm Copilot works before the day

Open VS Code, create a file, and check that you get inline suggestions and that Copilot Chat responds. If not, make sure you're signed in and your Copilot subscription is active.

Accounts & Access

Permissions to sort out ahead of time (especially on managed/enterprise accounts)

  • Copilot enabled for your account by your organization (if you use a Business/Enterprise plan).
  • Ability to create repositories and fork public repositories under your account.
  • Copilot coding agent available. Availability depends on your Copilot plan/admin settings.
  • GitHub Actions enabled on a repo you control, with permission to add workflows (needed for Path 5).

Per-Path Requirements

Extras to install only for the paths you'll follow (in addition to the core list)

Path 1 — Start Coding with AI

IDE-based exercises in VS Code, ending with building a small app.

  • VS Code with GitHub Copilot + Copilot Chat, signed in (core).
  • Ability to fork the sample repo and clone it with Git.
  • For the end-to-end app: a runtime for your chosen stack — Python (Flask), Node.js (Express), or Java (Spring Boot).
  • Optional: use GitHub Codespaces instead of a local setup.

Path 3 — Extended Capabilities

CLI, SDK, Agent Skills, Sub-Agents, and Memory.

  • GitHub CLI (gh) v2.x, installed and authenticated. cli.github.com
  • A supported shell — Bash, Zsh, Fish, or PowerShell.
  • For the SDK exercise: Node.js 18+ or Python 3.9+, plus ngrok (or similar) for HTTPS tunneling.
  • VS Code + Copilot Chat with agent mode for Skills & Sub-Agents.

Path 4 — Spec-Driven Development

Specs first, then GitHub Spec Kit.

  • Python 3.11+ and uv installed.
  • Git and a Copilot-enabled agent (VS Code or Copilot CLI).
  • The specify CLI — installed together during the session (no need to pre-install).

Path 5 — Agentic Workflows

Natural-language workflows with gh-aw.

  • GitHub CLI (gh), installed and authenticated.
  • A GitHub repository with Actions enabled and permission to add workflows.
  • A coding agent / model available to your repo (e.g., GitHub Copilot).
  • The gh-aw extension — installed together during the session.

Verify Your Setup

Run these the day before — each should succeed without errors

Open a terminal and check the tools you'll need. Not every command applies to every path — match them to the paths you're attending.

# Core / Paths 3 & 5 — GitHub CLI signed in
gh --version
gh auth status

# Core — Git
git --version

# Path 1 / Path 3 SDK — your chosen runtime(s)
node -v          # Node.js 18+ (Path 1 Node, Path 3 SDK)
python --version # Python 3.9+ (Path 3 SDK) / 3.11+ (Path 4)

# Path 4 — uv (provides the specify CLI installer)
uv --version

# Path 5 — confirm gh extensions can be listed
gh extension list
If gh auth status shows you're not logged in

Run gh auth login and follow the prompts. You should end up logged in to github.com.

Day-of Tips

Small things that prevent big delays

  • Sign in to github.com, VS Code, and the GitHub CLI ahead of time so you're not fighting auth prompts during exercises.
  • Update VS Code and the Copilot extensions to the latest version for the smoothest experience.
  • On a corporate network? Confirm access to github.com, the VS Code Marketplace, and npm/PyPI, or have a backup connection ready.
  • Short on time? Just complete the Core Prerequisites — the per-path extras can be installed during the session if needed.
